# Проектный практикум — АС «ServiceDesk»
## Автоматизированная система обработки заявок технической поддержки

---

## Тема проекта

**Название**: АС «ServiceDesk» — веб-приложение для регистрации, маршрутизации и обработки заявок внутренней технической поддержки организации.

**Суть**: Сотрудники организации создают заявки на решение IT-проблем (сломался принтер, нет доступа к системе, нужна установка ПО и т.д.). Специалисты технической поддержки принимают заявки в работу, меняют статусы, оставляют комментарии. Менеджер видит отчёты по нагрузке и SLA.

---

## Стек технологий

| Слой | Технология |
|------|-----------|
| Frontend | Vue 3 + Pinia + Vue Router + Vite |
| UI-библиотека | Naive UI (или Element Plus) |
| Backend | Go (Golang) + Gin (HTTP framework) |
| ORM / БД | GORM + PostgreSQL |
| Аутентификация | JWT (Access + Refresh токены) |
| API | REST JSON |
| Деплой (опционально) | Docker + Docker Compose |

---

## Роли пользователей

| Роль | Описание |
|------|---------|
| `employee` | Создаёт заявки, следит за статусом |
| `support` | Принимает, обрабатывает, закрывает заявки |
| `admin` | Управляет пользователями, категориями, приоритетами |

## Основные функции системы

### Для сотрудника (employee)
- Регистрация / вход по логину и паролю
- Создание новой заявки (тема, описание, категория, приоритет, вложение)
- Просмотр списка своих заявок с фильтрацией по статусу
- Просмотр детальной страницы заявки, переписка в комментариях
- Оценка качества решения после закрытия

### Для специалиста поддержки (support)
- Просмотр всех входящих заявок
- Назначение заявки на себя
- Смена статуса: `Новая → В работе → Ожидает пользователя → Решена → Закрыта`
- Добавление комментариев и вложений
- Фильтрация по приоритету, категории, дате

### Для администратора (admin)
- Управление пользователями (создание, блокировка, смена роли)
- Управление категориями и приоритетами заявок
- Просмотр дашборда: количество заявок по статусам, среднее время решения
