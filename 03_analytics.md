# Аналитические материалы — АИС «Контроль текущей успеваемости студентов»

---

## 1. Описание предметной области

### 1.1 Контекст

Контроль текущей успеваемости студентов — это процесс систематической регистрации, хранения и анализа результатов учебной деятельности студентов в течение семестра. Включает: ввод оценок за контрольные работы, лабораторные, практические занятия, посещаемость, рубежные и итоговые аттестации.

Без автоматизации типичные проблемы:
- Преподаватели ведут оценки в разрозненных таблицах Excel/бумажных журналах
- Студент узнаёт свои оценки только на занятии или в конце семестра
- Нет истории изменений — нельзя проследить, когда и кем была исправлена оценка
- Деканат получает сводные данные с задержкой и не видит общей картины
- Высокий риск потери данных при утере тетрадей/файлов преподавателя

### 1.2 Процесс «как есть» (As-Is)

```
Студент выполняет работу
        ↓
Преподаватель проверяет, ставит оценку в бумажный журнал
        ↓
Студент узнаёт оценку на следующем занятии (или нет)
        ↓
В конце семестра данные вручную переносятся в ведомость
        ↓
Деканат обрабатывает бумажные ведомости
```

### 1.3 Процесс «как должно быть» (To-Be)

```
Студент выполняет работу
        ↓
Преподаватель открывает веб-приложение
        ↓
Вводит оценку: дисциплина, тип работы, студент, балл, комментарий
        ↓
Студент видит оценку в личном кабинете сразу
        ↓
Деканат в реальном времени видит дашборд с успеваемостью
        ↓
При изменении оценки — запись в историю с обоснованием
```

## 2. Глоссарий

| Термин | Определение |
|--------|------------|
| Оценка (Grade) | Запись о результате учебной работы студента по конкретной дисциплине |
| Аттестация | Подтверждение достижения студентом минимального порогового балла |
| Дисциплина | Учебный предмет (курс), по которому ставятся оценки |
| Тип оценки | Классификация контрольного мероприятия: Текущая / Рубежная / Итоговая |
| Текущая оценка | Оценка за лабораторную, практическую работу, тест в течение модуля |
| Рубежная оценка | Контрольная точка в середине/конце модуля (рубежный контроль) |
| Итоговая оценка | Результат экзамена или зачёта по дисциплине |
| max_score | Максимально возможный балл за данный тип оценки |
| Статус оценки | Состояние записи: ожидает / оценено / аттестован / не аттестован |
| Преподаватель | Пользователь системы с правом выставления и редактирования оценок |
| Студент | Пользователь системы с правом просмотра своих оценок |
| Деканат / Администратор | Пользователь с правом управления системой и просмотра аналитики |
| JWT | JSON Web Token — механизм аутентификации |
| RBAC | Role-Based Access Control — разграничение прав по ролям |
| История изменений | Журнал всех изменений оценки: кто, когда, какое значение было и стало |

## 4. Подробные описания прецедентов

### UC-09 — Выставление оценки

| Поле | Значение |
|------|---------|
| **Актор** | Преподаватель |
| **Предусловие** | Пользователь авторизован с ролью `teacher`; дисциплина и студент существуют |
| **Основной поток** | 1. Преподаватель переходит в раздел «Очередь» или «Управление оценками» |
| | 2. Нажимает «Выставить оценку» |
| | 3. Заполняет форму: студент, дисциплина, тип оценки, название работы, балл, комментарий |
| | 4. Нажимает «Сохранить» |
| | 5. Система валидирует балл (0 ≤ балл ≤ max_score) и сохраняет запись |
| | 6. Запись отображается в личном кабинете студента |
| **Альтернативный поток** | 5а. Балл вне допустимого диапазона → система выводит ошибку валидации |
| **Постусловие** | Оценка сохранена; статус = `graded`; студент видит запись |

### UC-10 — Редактирование оценки

| Поле | Значение |
|------|---------|
| **Актор** | Преподаватель |
| **Предусловие** | Оценка существует и была создана этим преподавателем |
| **Основной поток** | 1. Преподаватель открывает детальную страницу оценки |
| | 2. Нажимает «Редактировать» |
| | 3. Изменяет балл, добавляет обязательный комментарий-обоснование |
| | 4. Нажимает «Сохранить изменения» |
| | 5. Система сохраняет новое значение и записывает в историю: старый балл, новый балл, автор изменения, время |
| **Постусловие** | Оценка обновлена; в истории появилась новая запись |

## 5. Схема базы данных (ERD)

### Таблицы

```sql
-- Пользователи
users
  id          UUID PRIMARY KEY
  email       VARCHAR(255) UNIQUE NOT NULL
  password    VARCHAR(255) NOT NULL           -- bcrypt hash
  full_name   VARCHAR(255) NOT NULL
  role        VARCHAR(20) NOT NULL            -- 'student', 'teacher', 'admin'
  is_active   BOOLEAN DEFAULT true
  created_at  TIMESTAMP
  updated_at  TIMESTAMP

-- Дисциплины (учебные предметы)
disciplines
  id          SERIAL PRIMARY KEY
  name        VARCHAR(100) NOT NULL           -- «Математический анализ», «СУБД», ...
  description TEXT
  created_at  TIMESTAMP

-- Типы оценок
grade_types
  id          SERIAL PRIMARY KEY
  name        VARCHAR(50) NOT NULL            -- «Текущая», «Рубежная», «Итоговая»
  color       VARCHAR(7)                      -- hex: #0d6efd
  max_score   INTEGER NOT NULL DEFAULT 100    -- максимально возможный балл

-- Записи об оценках
grades
  id            UUID PRIMARY KEY
  number        SERIAL UNIQUE                 -- человекочитаемый номер: #1001
  task_name     VARCHAR(255) NOT NULL         -- название работы / контрольного мероприятия
  notes         TEXT                          -- описание задания или условий
  status        VARCHAR(20) DEFAULT 'pending' -- pending, graded, certified, not_certified, closed
  student_id    UUID REFERENCES users(id)     -- кому выставлена оценка
  teacher_id    UUID REFERENCES users(id)     -- кто выставил
  discipline_id INTEGER REFERENCES disciplines(id)
  grade_type_id INTEGER REFERENCES grade_types(id)
  score         SMALLINT NULLABLE             -- выставленный балл
  created_at    TIMESTAMP
  updated_at    TIMESTAMP
  graded_at     TIMESTAMP NULLABLE            -- когда была оценена

-- Комментарии к оценке
comments
  id          UUID PRIMARY KEY
  grade_id    UUID REFERENCES grades(id) ON DELETE CASCADE
  author_id   UUID REFERENCES users(id)
  body        TEXT NOT NULL
  is_internal BOOLEAN DEFAULT false           -- true = только преподаватель видит
  created_at  TIMESTAMP

-- История изменений оценки
grade_history
  id          UUID PRIMARY KEY
  grade_id    UUID REFERENCES grades(id) ON DELETE CASCADE
  changed_by  UUID REFERENCES users(id)
  old_value   VARCHAR(20)                     -- предыдущий статус
  new_value   VARCHAR(20) NOT NULL            -- новый статус
  changed_at  TIMESTAMP DEFAULT NOW()

-- Refresh-токены
refresh_tokens
  id          UUID PRIMARY KEY
  user_id     UUID REFERENCES users(id)
  token       VARCHAR(255) UNIQUE NOT NULL
  expires_at  TIMESTAMP NOT NULL
  created_at  TIMESTAMP
```

### Связи

```
users ──< grades (student_id)        студент имеет много оценок
users ──< grades (teacher_id)        преподаватель выставляет много оценок
disciplines ──< grades               дисциплина объединяет много оценок
grade_types ──< grades               тип объединяет много оценок
grades ──< comments                  у оценки есть комментарии
grades ──< grade_history             у оценки есть история изменений
users ──< comments                   пользователь добавляет комментарии
users ──< grade_history              пользователь вносит изменения
```

## 6. REST API — описание эндпоинтов

### Аутентификация

| Метод | URL | Описание | Роли |
|-------|-----|---------|------|
| POST | `/api/v1/auth/register` | Регистрация нового пользователя | — |
| POST | `/api/v1/auth/login` | Вход, получение JWT | — |
| POST | `/api/v1/auth/refresh` | Обновление access-токена | authenticated |
| POST | `/api/v1/auth/logout` | Инвалидация refresh-токена | authenticated |

### Оценки (Grades)

| Метод | URL | Описание | Роли |
|-------|-----|---------|------|
| GET | `/api/v1/grades` | Список оценок (с фильтрами) | all |

| GET | `/api/v1/grades/:id` | Детали оценки + история изменений | all |
| PATCH | `/api/v1/grades/:id/status` | Выставить/изменить балл и статус | teacher, admin |
| PATCH | `/api/v1/grades/:id/assign` | Назначить преподавателя (взять в работу) | teacher |
| PATCH | `/api/v1/grades/:id/acknowledge` | Студент подтверждает ознакомление | student |

## 8. Формат JWT-токенов

### Access Token payload

```json
{
  "sub": "uuid пользователя",

  "role": "student",
  "exp": 1234567890
}
```

```
Authorization: Bearer <access_token>
```

## 9. Диаграмма состояний оценки

```
        [Запись создана]
               │
           pending (●)
               │  преподаватель выставляет балл
               ▼
            graded
           /        \
          ↓          ↓
   not_certified   certified
  (балл < порога)  (балл ≥ порога)
          \          │
           ↓         │ финальная итоговая аттестация
              closed (■)
```

Разрешённые переходы:

| Из | В | Кто |
|----|---|-----|
| pending | graded | teacher |
| graded | not_certified | teacher (рубежная/итоговая) |
| graded | certified | teacher (рубежная/итоговая) |
| not_certified | graded | teacher (пересдача) |
| graded | pending | teacher (отзыв оценки) |
| certified | closed | admin (завершение семестра) |
